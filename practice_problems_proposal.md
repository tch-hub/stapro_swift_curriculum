# 練習問題カリキュラム提案（Timer & ToDoList）

## 基本コンセプト
すべてのステップの練習問題において、以下のルールを統一します。
*   **アプリのカスタマイズではなく、学んだ知識の定着を図る**
*   **必ず「Xcodeで新規プロジェクトを作成（または前回の練習問題用プロジェクトを再利用）」して取り組む**
*   **各ステップの「核となる概念（コンポーネントや文法）」だけを抽出した汎用的なお題にする**

## タイマーアプリ

| ステップ | 学習テーマ | 練習問題 |
| :--- | :--- | :--- |
| **Step 1** | プロジェクトの作成とセットアップ | 練習問題なし |
| **Step 2** | UI構築 (VStack/HStack/Text/Button) | **「プロフィール画面」**<br>`VStack`と`HStack`を組み合わせて、アイコン（Textの代用でも可）、名前、横並びの「フォロー」「メッセージ」ボタンを持つプロフィール風UIを作成しましょう。 |
| **Step 3** | 時間選択ビューの実装 (Picker) | **「年月日（または果物）の選択リスト」**<br>`Picker` と `.pickerStyle(.wheel)` を使って、好きな果物や数字をスクロールして選べるセレクターを作成してみましょう。 |
| **Step 4** | 重ね合わせビュー (ZStack) | **「通知バッジ付きアイコン」**<br>`ZStack` と `Circle` を組み合わせて、プロフィールアイコンの右上に未読件数を示す赤い小さな丸（通知バッジ）を重ねて配置するレイアウトを作りましょう。 |
| **Step 5** | ViewModel / Timerロジック | **「シンプルな自動カウンター」**<br>`Observable` クラス（ViewModel）を作成し、`Timer.scheduledTimer` を利用して「1秒に1ずつ数字が増える」カウンターのロジックを作成しましょう。 |
| **Step 6** | ViewModelのUIへの反映 | **「シンプルなカウンター画面」**<br>数値を保持する `Observable` クラス（ViewModel）を作成し、`@StateObject` などでUIに紐づけ、画面に数字を表示して「＋１」「リセット」ボタンで数字を更新できるようにしましょう。 |
| **Step 7** | アラート機能 (.alert) | **「確認ダイアログ付きボタン」**<br>ボタンを押すと「本当に削除しますか？」といった内容のアラート（`.alert`）を表示し、「はい」「いいえ」を選択できるUIを作成しましょう。 |
| **Step 8** | カスタムビューへの切り出し | **「再利用できるカスタムボタン」**<br>タイトル（String）と色（Color）を受け取ってデザインされたボタンを返す `CustomButton` ビューを定義し、ContentViewでそれを3つ（赤・青・緑など）並べてみましょう。 |

## ToDoリストアプリ

| ステップ | 学習テーマ | 練習問題 |
| :--- | :--- | :--- |
| **Step 1** | プロジェクト作成 | 練習問題なし |
| **Step 2** | コンポーネント化 (引数付きView) | **「連絡先リストの1行（RowView）」**<br>アイコン（Image(systemName: "person.circle")など）と名前（Text）を横一直線に並べたコンポーネントを作りましょう。 |
| **Step 3** | リスト表示 (List / ScrollView) | **「ショッピングリスト」**<br>`List` 構造を使って、Array型の文字列データ（["りんご", "バナナ", "牛乳"]等）を繰り返し（`ForEach`）画面にリスト表示してみましょう。 |
| **Step 4** | 空状態の表現 (Image+Text) | **「エラー状態画面」**<br>`VStack`で警告アイコンと「通信エラーが発生しました」などのメッセージを中央に配置した、通信エラー時のような画面を作りましょう。 |
| **Step 5** | テキスト入力 (@Binding / TextField) | **「ログインフォーム」**<br>`TextField`を利用し、入力された文字を画面上の別の`Text`にリアルタイムで表示する入力画面を作りましょう。 |
| **Step 6** | ドロップダウンメニュー (Menu/Picker) | **「並び替えメニュー」**<br>`Menu` と `Picker` を使って、「新着順」「人気順」などを選択できるドロップダウンボタンを作り、選んだテキストが画面に表示されるようにしましょう。 |
| **Step 7** | データモデル (@Model) | **「本(Book)のデータ定義」**<br>SwiftData をインポートし、タイトル、著者名、既読状態(Bool)を持つ `Book` という `@Model` クラス（データモデル）を作成してみましょう。 |
| **Step 8** | 複雑なデータモデル | **「プレイリストのモデル」**<br>プレイリストの名前と、その中に含まれる「曲名（Stringの配列）」をプロパティとして持つ構造体を定義し、その内容を表示してみましょう。 |
| **Step 9** | 定数/初期データ | **「アプリ設定構造体（Config）」**<br>テーマカラーやシステムメッセージを `static let` で一元管理する `AppConfig` という構造体を作り、複数のViewからその定数を参照して表示してみましょう。 |
| **Step 10** | サービスの作成 (Logic分離) | **「ショッピングカート・サービス」**<br>「商品を配列に追加・削除する機能」を持ったサービスクラスを作り、ボタン経由でロジックを呼び出して総数を操作してみましょう。 |
| **Step 11** | データのフィルタリング・操作 | **「お気に入り絞り込み機能」**<br>配列データの中から「お気に入り(isFavorite = true)」のものだけを抽出（`.filter`）して返す処理を書き、それを画面に切り替えて表示してみましょう。 |
| **Step 12** | 管理画面・追加アクション | **「カテゴリ追加画面」**<br>TextFieldでの文字入力と「追加」ボタンを用意し、ボタンを押すと画面下のListの配列に新しい項目が追加されていくUIを作りましょう。 |
| **Step 13** | 複合レイアウトの組み立て | **「チャット画面のモックアップ」**<br>画面上部にタイトル、中央にList(メッセージ履歴)、一番下にTextFieldと送信ボタンという「よくある画面構成」をVStackで表現してみましょう。 |
| **Step 14** | データベース準備 (modelContainer) | **「アプリへのデータベース登録」**<br>任意の `@Model` クラス（例：`Memo`）を1つ定義し、それがアプリ全体で使えるように、エントリポイント（App）に `.modelContainer` を設定して起動できるようにしましょう。 |
| **Step 15** | 画面遷移 (NavigationStack) | **「詳細画面への遷移」**<br>`NavigationStack`と`NavigationLink`を使い、「設定」等のボタンを押したら「設定詳細画面(別のView)」に切り替わる動きを作ってみましょう。 |
| **Step 16** | 起動時の処理 (.onAppear / .task) | **「起動時の初期アクション」**<br>画面が表示された瞬間に`.onAppear`を使って「データ読み込み中…」から「読み込み完了」へテキストを切り替えるような処理を書いてみましょう。 |
| **Step 17** | DBデータの表示 (@Query) | **「データベースからの読み込み」**<br>任意の `@Model` クラス（例：`Memo`）を定義し、`@Query` を利用してそのデータを読み込み、`List` を使って画面に一覧表示させる画面を作成しましょう。 |
| **Step 18** | データ状態の更新 | **「いいねボタン（トグル）」**<br>データ項目にBool値を持たせ、`List` に表示された各項目の「ハート(Image)」をタップするたびに値が切り替わり、アイコンの見た目(塗りつぶし/縁取り)が変化するようにしましょう。 |
| **Step 19** | データベースへの追加 (context.insert) | **「新しいデータの登録」**<br>任意の `@Model` クラスを定義し、テキストフィールドに入力した文字をもとに、`context.insert()`を使って新しいデータをデータベースに保存するボタンを作りましょう。 |
| **Step 20** | スワイプ削除 (.onDelete) | **「リストのスワイプ削除」**<br>Listに対して `.onDelete` モディファイアを追加し、`context.delete()`を使って対象のアイテムをスワイプで消去できるようにしましょう。 |
| **Step 21** | 統合・発展課題 (アイデア検証) | 練習問題なし |
