## ビューのライフサイクル

SwiftUIのビューには、画面に表示されるとき（`onAppear`）と、画面から消えるとき（`onDisappear`）のタイミングで特定の処理を実行するためのモディファイアが用意されています。

```swift title="ContentView.swift"
import SwiftUI

struct ContentView: View {
    @State private var isShowing = true
    @State private var message = "待機中..."

    var body: some View {
        VStack(spacing: 20) {
            Toggle("ビューの表示切り替え", isOn: $isShowing)
                .padding()

            Text(message).foregroundColor(.gray)

            if isShowing {
                Text("対象のテキスト")
                    .font(.title)
                    .onAppear { message = "対象のテキストが表示されました！" }
                    .onDisappear { message = "対象のテキストが非表示になりました！" }
            }
            
            Spacer()
        }
        .padding()
    }
}

#Preview {
    ContentView()
}
```

これらのモディファイアは、画面が表示されたときに初期設定を行う処理や、画面が閉じるときにタイマーを停止する処理などによく使われます。

## 非同期処理 (task)

`onAppear` と似ていますが、データの通信処理（ネットワークからデータを取得するなど）を行う場合は `.task` モディファイアを使うのがより推奨されます。

```swift title="ContentView.swift"
import SwiftUI

struct ContentView: View {
    @State private var data = "読み込み中..."

    var body: some View {
        Text(data)
            .task {
                // 非同期処理をここに書くことができます
                try? await Task.sleep(nanoseconds: 2_000_000_000) // 2秒待つ（時間のかかる処理の例）
                data = "データの読み込みが完了しました！"
            }
    }
}

#Preview {
    ContentView()
}
```

`.task` は、ビューが表示されると自動的に実行され、ビューが消えると自動的にキャンセルされるという強力な特徴があります。これにより、裏側での無駄な処理が残り続けることを防ぐことができます。

---

## onAppear と task の違い

`onAppear` と `.task` はどちらも「画面が表示されたとき」に実行されますが、機能面で明確な違いがあります。

- **`onAppear`**: 画面が表示されたタイミングで、**すぐに終わる処理**（初期値のセットや簡単な計算など）を行うのに向いています。ブロック内で直接非同期処理（`await` を伴う処理）を書くことはできません。

```swift title="onAppearの例"
Text("Hello")
    .onAppear {
        // すぐに終わる処理（問題なく動く）
        print("表示されました")
        
        // エラーになる書き方（awaitは使えない）
        // await Task.sleep(nanoseconds: 1_000_000_000) 
    }
```

- **`.task`**: 画面が表示されたタイミングで、**時間がかかる処理**（ネットワーク通信によるデータの取得など）を行うのに向いています。ブロック内で `await` を自然に使うことができ、さらに**画面が閉じられた瞬間に実行中の処理が自動的にキャンセル（中断）される**という優れた安全機能を持っています。

```swift title="taskの例"
Text("Hello")
    .task {
        // 時間がかかる処理（awaitが使える）
        print("読み込み開始...")
        try? await Task.sleep(nanoseconds: 2_000_000_000) // 2秒待つ
        print("読み込み完了！")
    }
```

基本的には、「時間がかかる処理（非同期処理）には `.task` を使い、それ以外は `onAppear` を使う」と覚えておきましょう。

## 状態の変化を監視する (onChange)

ユーザーの入力や何らかの処理によって `@State` などの変数の値が変わったことを検知して、追加の処理を行いたい場合は `.onChange` モディファイアを使います。

```swift title="ContentView.swift"
import SwiftUI

struct ContentView: View {
    @State private var searchText = ""
    @State private var result = ""

    var body: some View {
        VStack {
            TextField("検索キーワード", text: $searchText)
                .textFieldStyle(.roundedBorder)
                .padding()
                // searchTextの値が変化するたびに実行される
                .onChange(of: searchText) { oldValue, newValue in
                    result = "「\(newValue)」を検索中..."
                }

            Text(result)
        }
    }
}

#Preview {
    ContentView()
}
```

iOS 17以降の新しいSwiftUIでは、`.onChange(of:変数名)` のクロージャは `oldValue`（変更前の古い値）と `newValue`（変更後の新しい値）の2つの引数を受け取る形で書くのが標準になっています。これを使うことで、「入力されるたびに文字数をチェックする」などの連動した処理が可能になります。
