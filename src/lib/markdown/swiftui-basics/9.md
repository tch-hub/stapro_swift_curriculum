## NavigationStackの基本

SwiftUIで別の画面に遷移するには、`NavigationStack`と`NavigationLink`を組み合わせます。

```swift title="ContentView.swift"
import SwiftUI

struct ContentView: View {
    var body: some View {
        NavigationStack {
            VStack(spacing: 20) {
                Text("最初の画面")
                    .font(.largeTitle)
                
                NavigationLink("次の画面へ進む") {
                    SecondView()
                }
                .buttonStyle(.borderedProminent)
            }
            .navigationTitle("ホーム")
        }
    }
}

struct SecondView: View {
    var body: some View {
        VStack {
            Text("2番目の画面です")
                .font(.title)
        }
        // 遷移先の画面でもタイトルを設定できる
        .navigationTitle("詳細ページ")
        // タイトルの表示スタイルを小さくする
        .navigationBarTitleDisplayMode(.inline)
    }
}

#Preview {
    ContentView()
}
```

このコードでは、以下の要素が使われています。

- **`NavigationStack`**: 画面遷移を管理するための最も大枠のコンテナです。画面遷移を行う場合は、階層の一番外側にこれを1つだけ配置します。
- **`NavigationLink`**: タップすると別の画面に遷移するボタンのような役割を持ちます。遷移先のビュー（ここでは`SecondView()`）を指定します。
- **`.navigationTitle`**: ナビゲーションバーにタイトルを表示するためのモディファイアです。

## 値を渡しながら遷移する (navigationDestination)

より実践的なアプリでは、「リストから選んだ項目の詳細画面」のように、選択したデータに合わせて遷移先の画面を変化させることがよくあります。
iOS 16以降のSwiftUIでは、`navigationDestination(for:destination:)` を使う方法が推奨されています。

```swift title="ContentView.swift"
import SwiftUI

// 渡したいデータの型（Hashable準拠）
struct Fruit: Hashable {
    let name: String; let color: Color
}

struct ContentView: View {
    let fruits = [Fruit(name: "りんご", color: .red), Fruit(name: "バナナ", color: .yellow)]

    var body: some View {
        NavigationStack {
            List(fruits, id: \.name) { fruit in
                NavigationLink(fruit.name, value: fruit) // 値を渡す
            }
            .navigationTitle("フルーツ図鑑")
            .navigationDestination(for: Fruit.self) { fruit in
                FruitDetailView(fruit: fruit) // 受け取った値で遷移
            }
        }
    }
}

struct FruitDetailView: View {
    let fruit: Fruit
    var body: some View {
        ZStack {
            fruit.color.opacity(0.3).ignoresSafeArea()
            Text("\(fruit.name)の詳細").font(.largeTitle)
        }
        .navigationTitle(fruit.name)
    }
}

#Preview {
    ContentView()
}
```

この方法では、`NavigationLink` の `value` にデータを渡し、`.navigationDestination` でそのデータを受け取って対応するビューを表示します。データとビューの表示を明確に分離できるため、複雑なアプリでも画面遷移の管理がしやすくなります。
