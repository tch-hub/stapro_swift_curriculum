# ステップ1: アプリ企画・設計・プロジェクト作成

今日からいよいよ、あなただけのオリジナルアプリを作ります！  
まずXcodeでプロジェクトを作成し、その中に計画書（PLAN.md）を作ってから、アイデアを固めていきます。

> 🖥️ **作るのは macOS アプリです。**  
> iPhoneアプリは無料の開発者アカウントだと7日ごとにMacと繋いで再インストールが必要ですが、macOSアプリなら制限なく動かし続けられます。SwiftUI・SwiftDataはiPhoneと全く同じコードで書けます！

---

## 1. Xcodeでプロジェクトを作成する

1. Xcodeを起動し、「Create a new Xcode project」を選択
2. テンプレート選択画面で上部タブを **「macOS」** に切り替えてから「**App**」を選択
3. 以下のように設定する

| 項目 | 内容 |
|------|------|
| Product Name | アプリ名（仮でOK。例: MyApp） |
| Team | 自分のApple ID |
| Organization Identifier | com.yourname |
| Interface | SwiftUI |
| Language | Swift |

> ⚠️ **「iOS」タブのままだとiPhoneアプリになります。必ず「macOS」タブを選んでください！**

---

## 2. PLAN.md を作成する

プロジェクトの中に計画書ファイルを用意します。

1. Xcodeのプロジェクトナビゲーター（左側）でプロジェクト名を右クリック
2. 「New File from Template...」→「Other」→「Empty」を選択
3. ファイル名を `PLAN.md` として保存

以下のテンプレートをコピーして `PLAN.md` に貼り付けましょう。

```markdown
# アプリ企画書

## アプリ名
（決まったら書く）

## 目的
- 誰のために:
- 解決する問題:

## 画面一覧
-

## 保存するデータ（ゲームの場合は不要）
-

## Must機能（絶対に作る）
-

## Want機能（余裕があれば）
-
```

> 💡 `PLAN.md` はXcodeのナビゲーターから常に参照できます。これから決めた内容をここに書き込んでいきましょう。

---

## 3. 何を作るか決める

### アプリテーマの例
まだアイデアが思い浮かばない場合はこんなアプリはどうでしょう？

**📱 ツール・記録系**
- **習慣トラッカー**: 毎日やりたいことの記録（運動・勉強・読書など）
- **単語帳アプリ**: 英単語や用語を覚えるフラッシュカード
- **食事記録アプリ**: 食べたものをメモする
- **目標管理アプリ**: 目標を設定して達成度を記録する
- **メモアプリ**: カテゴリ別にメモを整理する
- **勉強タイマー**: 教科ごとに時間を記録するタイマー

**🎮 ゲーム系**（`/project/original-app` のアイデア例でプレイ可能）
- **2048**: 数字のスライドパズル
- **15パズル**: タイル並べ替えゲーム
- **神経衰弱**: カードのめくり合わせゲーム
- **ライツアウト**: ライトを全部消すパズル
- **オリジナルパズル**: ルールを自分で考えたゲーム

> 💡 ゲームを作る場合も、SwiftDataを使うと面白い機能が実現できます！
> - **ハイスコアの保存**: ゲームを閉じてもスコアが残る
> - **プレイ履歴**: 過去のプレイ結果を一覧表示
> - **設定の保存**: 難易度・音量などをアプリ再起動後も維持
>
> 「まずは`@State`だけで動かして、後からSwiftDataを追加する」という進め方もOKです。

決まったら `PLAN.md` の **アプリ名** と **目的** を埋めましょう。

---

## 4. 必要な画面を書き出す

アプリに必要な画面を書き出しましょう。

**例）習慣トラッカーの場合**
```
- ホーム画面（習慣の一覧＋今日の達成状況）
- 習慣作成画面（名前・頻度を入力）
```

> 💡 まずは2〜3画面でOK！多すぎると完成しなくなります。

決まったら `PLAN.md` の **画面一覧** を埋めましょう。

---

## 5. 保存するデータを決める

アプリで「記憶しておきたい情報」を書き出しましょう。

**例）習慣トラッカーの場合**
```
- 習慣のタイトル（文字列）
- 達成した日付のリスト
- 作成日時
```

**例）ゲームの場合**
```
- ハイスコア（数値）
- プレイ日時（日付）
- 難易度設定（文字列 or 数値）
```

> 💡 ヒント: ToDoリストアプリでは `ToDoTask`・`ToDoTab` というデータモデルを作りました。  
> 検索バーで「@Model」と検索すると実装例を確認できます。

決まったら `PLAN.md` の **保存するデータ** を埋めましょう。

---

## 6. 機能の優先順位を決める

| 優先度 | 機能 |
|--------|------|
| ⭐ Must（絶対必要） | 例：習慣の追加・一覧表示・完了チェック |
| 🔵 Want（余裕があれば） | 例：カレンダー表示・通知機能 |

> Must機能だけで動くアプリが作れれば大成功です！

決まったら `PLAN.md` の **Must機能 / Want機能** を埋めて完成です！
