## @Stateを使った状態管理

@StateはSwiftUIでビューの状態を管理するための特別なプロパティラッパーです。

<img src="/images/swiftui-basics/4_1.png" alt="@Stateを使った状態管理" class="mobile-screenshot" />

```swift title="ContentView.swift"
import SwiftUI
struct ContentView: View {
    @State private var count = 0 // カウントを保存する状態変数
    
    var body: some View {
        VStack { // 縦方向にビューを並べる
            Text("カウント: \(count)") // 現在のカウントを表示
                .font(.largeTitle) // 大きなフォントサイズ
            
            HStack { // 横方向にボタンを並べる
                Button(action: {
                    count -= 1 // カウントを1減らす
                }) {
                    Text("-") // マイナスボタンのラベル
                        .font(.title) // タイトルサイズのフォント
                        .frame(width: 50, height: 50) // ボタンのサイズ
                        .background(Color.red) // 赤い背景
                        .foregroundColor(.white) // 白い文字
                        .cornerRadius(25) // 角を丸くする
                }
                
                Button(action: {
                    count += 1 // カウントを1増やす
                }) {
                    Text("+") // プラスボタンのラベル
                        .font(.title) // タイトルサイズのフォント
                        .frame(width: 50, height: 50) // ボタンのサイズ
                        .background(Color.green) // 緑の背景
                        .foregroundColor(.white) // 白い文字
                        .cornerRadius(25) // 角を丸くする
                }
            }
        }
    }
}

#Preview {
    ContentView() // プレビューを表示
}
```

インタラクティブなUIを作成する際に欠かせない重要な概念です。

```swift
@State private var count = 0 // カウントを保存する状態変数
```

通常の変数とは異なり、@Stateがついた変数の値が変更されると、その変数を使っているビューが自動的に再描画されます。これにより、ユーザーの操作に応じてUIがリアルタイムに更新されます。

## @Stateの仕組み

SwiftUIでは、UIはデータ（状態）の反映として描画されます。ユーザーがボタンをタップしたり、テキストを入力したりすると、状態が変化し、それに応じてUIが更新される必要があります。

@Stateは、この状態変化をSwiftUIに通知するための仕組みです。@Stateがついた変数の値が変わると、SwiftUIは自動的にその変数を使っているビューを再描画します。これを「リアクティブ（反応的）プログラミング」と呼びます。

## なぜ@Stateが必要なのか

SwiftUIのビューはstruct（構造体）でできており、通常の変数は変更できません。しかし、インタラクティブなUIでは状態を変更する必要があります。@Stateはこの問題を解決します。

```swift
struct ContentView: View {
    // ❌ 通常の変数は変更できない
    // var count = 0  // エラーになる
    
    // ✅ @Stateなら変更可能
    @State private var count = 0
    
    var body: some View {
        VStack {
            Text("カウント: \(count)")
            Button("カウントアップ") {
                count += 1  // ここで状態が変更され、UIが自動更新される
            }
        }
    }
}
```

## @Stateのルール

1. **private varとして宣言する**
   - @State変数は必ず`private var`として宣言します
   - これにより、変数がそのビュー内でのみ変更可能になり、外部からの意図しない変更を防ぎます

2. **初期値を設定する**
   - @State変数には必ず初期値を設定します
   - ビューが初めて表示される時の状態を定義します

3. **$記号でバインディング**
   - TextFieldやToggleなどのコントロールにバインドする場合は、変数名の前に`$`記号を付けます
   - `$count`のように書くと、コントロールの値と@State変数が同期されます

```swift
@State private var name = ""
@State private var isOn = false
@State private var volume = 0.5

var body: some View {
    VStack {
        // TextFieldでは$nameのように$記号を使う
        TextField("名前を入力", text: $name)
        
        // Toggleでも$isOnのように$記号を使う
        Toggle("オン/オフ", isOn: $isOn)
        
        // Sliderでも$volumeのように$記号を使う
        Slider(value: $volume, in: 0...1)
    }
}
```

## @Stateの使用例

このカウンターアプリの例では、`count`という@State変数を使ってカウンターの値を管理しています。

- `+`ボタンをタップすると`count += 1`が実行されます
- `-`ボタンをタップすると`count -= 1`が実行されます
- `count`の値が変わるたびに、`Text("カウント: \(count)")`が自動的に更新されて新しい値が表示されます

```swift
Button(action: {
    count += 1  // countの値が変わると、UIが自動的に更新される
}) {
    Text("+")
}
```

## @State vs 通常の変数

| 通常の変数 | @State変数 |
|------------|-------------|
| 値の変更がUIに反映されない | 値の変更がUIに自動反映される |
| struct内で変更不可能 | @Stateで変更可能 |
| メモリ管理がシンプル | SwiftUIが自動管理 |

## @Stateの注意点

- @State変数はビューが再作成されると初期値に戻ります
- 複雑な状態管理が必要な場合は@StateObjectやEnvironmentObjectを使います
- @Stateはローカルな状態管理に適しています

@StateはSwiftUIのリアクティブプログラミングの基礎となるもので、これを使うことで複雑な状態管理をシンプルに実装できます。カウンターアプリのようなシンプルなものから、複雑なフォームまで、様々なインタラクティブなUIの基礎となります。
